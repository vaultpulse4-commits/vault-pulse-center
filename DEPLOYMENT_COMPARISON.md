### Step-by-Step Setup (15 menit)

#### 1. Buat Railway Account
```
1. Go ke: https://railway.app
2. Sign up dengan GitHub
3. Authorize Railway
```

#### 2. Create New Project
```
1. Click "New Project"
2. Pilih "Deploy from GitHub repo"
3. Select repository: vault-pulse-center
4. Authorize Railway to access GitHub
```

#### 3. Add PostgreSQL Database
```
1. Di project dashboard, click "New"
2. Pilih "Database" â†’ "PostgreSQL"
3. Railway automatically akan create DATABASE_URL
4. Copy DATABASE_URL (you'll need it)
```

#### 4. Configure Backend Variables
```
1. Select service "vault-pulse-server"
2. Buka "Variables" tab
3. Add variables:
   DATABASE_URL=<auto-generated>
   NODE_ENV=production
   PORT=3001
   FRONTEND_URL=https://your-domain.com
   (akan update nanti setelah frontend deploy)
```

#### 5. Deploy Backend
```
1. Railway akan auto-detect pake root directory: server/
2. Build command: npm install && npm run build && npm run prisma:generate
3. Start command: npm start
4. Klik deploy
5. Wait ~5 menit untuk build & start
6. Get backend URL dari Railway (e.g., railway.app/xxx)
```

#### 6. Deploy Frontend (Vercel)
```
1. Go ke: https://vercel.com
2. "New Project" â†’ Import vault-pulse-center repo
3. Root Directory: ./ (tidak ada)
4. Build Command: npm run build (atau bun run build)
5. Output Directory: dist
6. Add Environment Variable:
   VITE_API_URL=https://api.your-domain.com
7. Deploy
8. Get Vercel URL (e.g., vault-pulse-center.vercel.app)
```

#### 7. Setup Custom Domain
```
Railway Backend:
1. Railway â†’ Settings â†’ Domains
2. Add custom domain: api.your-domain.com
3. Point DNS: A record to Railway IP

Vercel Frontend:
1. Vercel â†’ Settings â†’ Domains
2. Add custom domain: your-domain.com
3. Follow DNS configuration
4. Add www.your-domain.com juga
```

#### 8. Update Backend FRONTEND_URL
```
1. Back ke Railway
2. Backend variables
3. Update FRONTEND_URL=https://your-domain.com
4. Save & deploy
```

#### 9. Test
```
1. Open https://your-domain.com
2. Check browser console â†’ API connection OK?
3. Try login
4. Check Network tab â†’ API calls working?
5. curl https://api.your-domain.com/health
   Should return: {"status":"ok","timestamp":"..."}
```

## ğŸ†˜ TROUBLESHOOTING

### "Frontend tidak bisa konek ke API"
```
1. Check VITE_API_URL environment variable
2. Check API backend running
3. Check CORS settings in backend
4. Check network tab in browser
```

### "Database migration error"
```
1. Check DATABASE_URL format
2. Verify database user permissions
3. Run migrations manually
4. Check logs untuk detail error
```

### "Domain tidak mengarah ke aplikasi"
```
1. Check DNS records propagated
2. Verify A record pointing correct IP
3. Check Nginx/reverse proxy config
4. Wait 24 hours untuk full propagation
```

### "SSL certificate error"
```
1. Verify certificate installed
2. Check certificate valid date
3. Clear browser cache
4. Try different browser
```
